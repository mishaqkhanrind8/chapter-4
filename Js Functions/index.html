JavaScript Functions

functions are important in JavaScript and act like building blocks.

they perform tasks or calculations based an input and output.

for a procedure to be a function, it should take input and produce output.

functions need to be defined where they are going to be used,

they help avoid repeating the same code and make programs easier to manage.

functions allow breaking down big programs into smaller, more manageable ports.

function syntax

a JavaScript functions is defined with the function keyword, followed by a name, followed by parentheses().

function names can contain letters, digits, underscores and dollar signs (same rules as variables),

the parentheses may include parameter names separated by commas:

(parameter1, parameter2,...)

the code to be executed, by the function, is placed inside curly brackets{}

function name (parameter1, parameter2, parameter3) {
// code to be executed
}
// function to compute the product of p1 and p2

function myfunction (p1, p2) {
console.log(p1 * p2);
}

calling function

function can be called , or executed, elsewhere in code using parentheses following the function name.

when a function is called, the code inside its function body runs.

// defining the function
function sum(num1, num2) {
return num1 + num2;
}
// calling the function
console.log(sum(2, 4));
// output 6

parameter vs argument

parameter: Variables in the function declaration that act as placeholders for values that the function expects to receive when it's called.

Arugment: Actual values passed to the function when it is called.

function greet(name) { // 'name' is a parameter
console.log("hello", + name + "!" );
}
greet("ishaq-khan"); // "ishaq-khan" is an argument

default values

functions can also be defined with default values for one, or all of the parameters,

if no arguments are passed the default values are used, if arguments are incluted they will overirde the default values.

function return

when a function is called, it executes its code and evaluates a result, which by default is undefined

to capture and return a value from a function. the return keyword is used.

once a function finds a return statement, it stops working and hands back whatever it found.

the return keyword allows functions to produce an output, which can be stored in a variable for further use,

without return

function add(a, b) {
let sum = a + b;
}
let result = add(3, 5);
console.log(result);
// output : undefined

with return

function add(a, b) {
return a + b;
}
let result = add(3, 5);
console.log(result);
// output : 8

function experssion

A JavaScript function expression is a way to define a function in JavaScript by assigning it to a variable.

Unlike function declarations, where the function name is mandatory, function expressions allow you to create anonymous functions or to assign a name to the function, here's an example of a function expression:

let add = function(x, y) {
return x + y;
}

In this example, add is a variable that holds an anonymous function.

This function takes two parameters (x and y) and returns their sum.

Since it's assigned to a variable, you can use the add variable to call this function later in your code:

console.log(add(3, 5)); // output is : 8

function scope

variables declared outside of any function have global scope.

they can be accessed from anywhare in the code, including inside functions.

global variables are accessible throughout athe entire program.

variables declared whtin a function have local scope.

they are accessible only whithin that function and not outside of it.

local variables have a limited lifespan and are destoryed once the function finishes executing.
