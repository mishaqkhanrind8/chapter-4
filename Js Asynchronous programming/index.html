Asynchronous programming

Asynchronous programming lets JavaScript do multiple things at once. making apps faster and smoother.

In JavaScript, asynchronous programming is crucial for handling operations that may take time, such as fetching data from a server or reading a file.

It's like cooking while waiting for water to boil, you don't just store at the pot, you do other tasks too!

difference between synchronous and asynchronous

Synchronous code executes line by line, blocking further execution until each line finishes.

Aynchronous code allows multiple tasks to be executed simultaneously without blocking the main thread (like fetching data).

callback functions

Callback functions are like leaving a note for someone to call you back when they're done with a task.

They're used in JavaScript to handle tasks that take time. Like leading data from a server.

function myFunction(callback) {
setTimeout(() => {
const data = { name: "Aman", age: 21};
callback(data);
}, 3000);
}
myFuction((data) => {
console.log("data: ", data);
} );

promises

Promises are like a promise to do something, but you don't know when it will be done.

They're great for handling tasks that may succeed or fail, like fetching data from a server.

function mydata() {
return new promise((resolve, reject) => {
setTimeout(() => {
const data = {name: "Ishaq", age; 23};
resolve(data);
}, 2000);
});
}
mydata()
.then((data) => console.log("Data", data))
// output: data: {name: "Ishaq" , age: 23}
.catch((error) => console.error("Error", error));

async/Await

Async functions enable writing asynchronous code in a synchronous-like manner using the async and await keywords.

async makes a function return a promise

await makes a function wait for a promise

function fetchData() {
return new promise((resolve, reject) => {
setTimeout(()=> {
resolve("Data received");
}, 2000);
});
}
fetchData()
.then(data => console.log(data))
// output: Data received (after 2 seconds)
.catch(error => console.error(error));

resolve and reject

the two arguments (resolve and reject) are pre-defined by JavaScript.

We will not create them, but call one of them when the executor function is ready.

Very often we will not need a reject function.
