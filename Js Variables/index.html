javaScript variables

variables are containers for storing data values.

variables are used to store information to be referenced and manipulated in a program.

A variable is assigned a value in one place and then used repetitively.

you can declare a variable using the var, let, or const keyword.

The var keyword was used in all JavaScript code from 1995 to 2015.

the let and const keywords were added to JavaScript in 2015 to nowadays

The var keyword should only be used in code written for older browsers.

JavaScript variables can hold numbers like 100 and text values like "smash code"

in programming, text values are called text strings.

javaScript can handle many types of data, but for now, just think of numbers and strings.

strings are written inside double or single quotes. Numbers are written without quotes.

If you put a number in quotes, it will be treated as a text string.

declaring a variable

You can declare and initialize in a single line or you can do that in two lines

You can declare many variables in one statement. start the statement with let and separate the variables by comma

As with algebra, you can do arithmetic with JavaScript variables, using operators like - and +:

you can also add strings, but strings will be concatenated.

javaScript let

the let keyword was introduced in ES6 (2015)

variables declared with let have bock scope

variables declared with let must be declared before use.

variables declared with let cannot be redeclared in the same scope

block scope

variables declared inside a { } block cannot be accessed from outside the block

global scope

variables declared with the var always have global scope

variables declared with var keyword can NOT have block scope

redeclaring variable

vairables defined with let can not be redeclared.

variables defined with var can be redeclared.

redeclaring variable using var

redeclaring a variable using the var keyword can impose problums.

redeclaring a variable inside a block will also redeclare the variable outside the block

hoisting

Hoisting is a javaScript mechanism where variables and function declarations are moved to the top of their scope before code execution.

so var variables are hoisted to the top of their scope and initialized with a value of undefined.

just like var, let and const declarations are hoisted ot the top.

unlike var which is initialized as undefined. the let and const keywords are not initialized.

so if you try to use a let or const variable before declaration, you'll get a reference Error.

javaScript const

the conts keyword was introduced in ES6 (2015)

variables defined with const cannot be redeclared.

variables defined with conts cannot be reassigned.

variables defined with const have block scope

reassigning variable

a const variable cannot be reassigned

variables with var or let can be reassigned

javaScript const variables must be assigned a value when they are declared

when to use JavaScript const ?

Always declare a variable with const when you know that the value should not be changed.

use const when you declare:

  • a new Array
  • a new Object
  • a new Function
  • a new RegExp
